🔐 SSH-Docs

Browse local documentation over SSH with a small read-only shell.

Python 3.8+ MIT License asyncssh + click

SSH-Docs starts an SSH server for your docs and lets users explore files with familiar commands like ls, cd, cat, find, grep, head, tail, pwd, and help.

Install from PyPI View on GitHub

Why SSH-Docs?

It gives terminal-first teams a simple way to expose documentation over SSH without building a web UI.

📖

Read-only browsing

Users can navigate and inspect files without editing content.

🧭

Familiar commands

Browse docs with Unix-style commands instead of learning a custom interface.

⚡

Low setup

Start with defaults, or add a YAML config file only when you need more control.

🔐

Flexible auth

Run in public mode or require a password or authorized SSH keys.

Quick Start

Install the package, serve a directory, then connect with any SSH client.

pip install ssh-docs

# Serve an auto-detected docs directory, or the current directory if none is found
ssh-docs serve

# Serve a specific directory
ssh-docs serve ./docs

# Use a custom port
ssh-docs serve ./docs --port 3000

# Connect
ssh localhost -p 2222

Available Shell Commands

The interactive shell intentionally stays small and focused on browsing.

pwd, ls, cd, cat, head, tail, find, grep, help, exit
/site$ ls
/site$ cd guides
/site$ cat getting-started.md
/site$ grep -R "authentication" .
/site$ find . -name "*.md"
/site$ head -n 20 README.md
/site$ tail -n 10 changelog.txt
/site$ pwd
/site$ help
/site$ exit

CLI Commands

ssh-docs serve

Start the SSH server for a documentation directory.

Options: --port, --site-name, --config, --host, --auth, --keys-file, --password, --no-config, --log-level

ssh-docs init

Create a .ssh-docs.yml configuration file.

ssh-docs validate

Validate config syntax and inspect resolved settings.

ssh-docs keygen

Generate SSH host keys for the server.

ssh-docs completion

Generate shell completion scripts for Bash, Zsh, or Fish.

Configuration Highlights

SSH-Docs can run without config, but a YAML file enables more control.

Core settings

site_name, content_root, port, and host.

Authentication

Set auth.type to public, password, or key.

Features

Feature flags exist for syntax highlighting, line numbers, and search indexing.

Rate limiting

Control connection and failed-auth thresholds in the rate_limiting section.

site_name: "My Project Documentation"
content_root: "./docs"
port: 2222
host: "0.0.0.0"

auth:
  type: "public"

server:
  max_connections: 10
  timeout: 300
  log_level: "info"

features:
  syntax_highlighting: false
  line_numbers: false
  search_index: false

rate_limiting:
  enabled: true
  max_connections_per_ip: 3
  max_connections_per_minute: 10
  max_failed_auth_attempts: 5
  failed_auth_window_seconds: 300
  max_total_connections: 100

Auto-Detection

Content root

SSH-Docs checks docs/, documentation/, public/, and dist/ before falling back to the current directory.

Site name

SSH-Docs reads package.json, then pyproject.toml, and finally the current directory name.
